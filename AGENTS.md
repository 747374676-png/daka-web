# 奖励打卡应用 - 需求拆解文档

## 产品概述

- **产品类型**: 个人效率工具/习惯追踪应用
- **场景类型**: <scene_type>prototype-app</scene_type>
- **目标用户**: 希望通过奖惩机制管理日常行为的个人用户
- **核心价值**: 通过每日打卡和余额激励，帮助用户养成好习惯、纠正坏习惯
- **界面语言**: 中文
- **主题偏好**: 浅色（清新配色风格）
- **导航模式**: 路径导航
- **导航布局**: Topbar（消费者前台工具，非中后台管理系统）

## 功能列表

### 页面: 用户标识页 (`/login`)

- **页面目标**: 用户设置或输入自己的标识（用户名/同步码），实现数据隔离和多设备同步
- **功能点**:
  - **首次使用引导**: 页面检测 localStorage 中无用户标识时，展示"设置你的同步码"表单（输入框 + 确认按钮），提示用户设置一个唯一的用户名或同步码
  - **已有用户登录**: 页面检测 localStorage 中有用户标识时，展示"欢迎回来，[用户名]" + "继续打卡"按钮（跳转 `/`）+ "切换账号"按钮
  - **切换用户**: 提供"使用其他同步码"入口，输入新的同步码后覆盖 localStorage 中的标识，从云端加载新用户数据，跳转 `/`
  - **数据加载**: 用户确认标识后，从云端数据库加载该标识对应的所有数据（事项列表、打卡记录、初始余额），存储到应用状态中

---

### 页面: 主打卡页 (`/`)

#### 区块: 顶部余额卡片

- **页面目标**: 展示当前余额和今日收益，提供核心激励反馈
- **功能点**:
  - **当前余额展示**: 大字号显示实时计算的当前余额（初始余额 + 所有奖励 - 所有惩罚），金额变化时带过渡动画
  - **今日收益展示**: 显示今日的奖励总额、惩罚总额、净收益（今日净值），正数绿色、负数红色
  - **初始余额设置**: 点击余额卡片右上角设置图标，弹出 Dialog 输入初始余额数值，保存后重新计算当前余额

#### 区块: 日期切换导航

- **页面目标**: 切换查看/操作不同日期的打卡记录
- **功能点**:
  - **左右箭头切换**: 点击左箭头切换到前一天，右箭头切换到后一天，不允许切换到未来日期
  - **日期选择器**: 点击日期文字弹出日期选择器（日历组件），可选择任意历史日期
  - **今日快捷按钮**: 当查看非今日日期时，显示"回到今天"按钮，点击跳转到今日
  - **日期显示**: 显示当前选中日期（格式：YYYY年MM月DD日 周X），若为今日则标注"今天"

#### 区块: 任务列表

- **页面目标**: 展示当前日期生效的任务事项，支持打卡操作
- **功能点**:
  - **任务过滤**: 根据当前选中日期，过滤出在该日期范围内生效的事项（开始日期 ≤ 当前日期 ≤ 结束日期），每日重复事项始终生效
  - **打卡按钮**: 每个事项右侧显示打卡按钮，未打卡状态为空心圆圈（灰色），已打卡状态为实心勾选圆圈（绿色），点击切换状态
  - **打卡逻辑**:
    - 奖励事项标记完成 → 余额增加对应金额，记录打卡条目（完成=true，金额变化=+金额）
    - 惩罚事项标记未完成 → 余额扣除对应金额，记录打卡条目（完成=false，金额变化=-金额）
    - 同一事项同一天可多次切换状态，每次切换更新对应打卡条目
  - **撤销打卡**: 顶部或任务列表提供"撤销上一次打卡"按钮，撤销最近一次打卡操作（恢复余额 + 删除对应打卡条目），支持连续撤销
  - **重置打卡**: 每个事项长按或右键菜单提供"重置今日打卡"选项，清除该事项今日所有打卡记录并重新计算余额
  - **空状态**: 当前日期无生效任务时，显示空状态插图和"暂无任务，点击下方按钮添加"提示

#### 区块: 余额趋势图

- **页面目标**: 可视化展示余额变化趋势
- **功能点**:
  - **时间范围切换**: 提供"7天"/"30天"切换按钮，默认显示最近7天
  - **折线图展示**: 以当前日期为终点，向前取7天或30天的每日余额净值，绘制折线图，X轴为日期、Y轴为余额
  - **汇总指标**: 图表下方显示累计获得奖励总额、累计惩罚总额、净收益三个汇总数字
  - **图表交互**: hover 数据点显示当日余额和日期

#### 区块: 历史记录（可折叠）

- **页面目标**: 查看每日打卡记录明细
- **功能点**:
  - **折叠展开**: 默认收起状态，点击"历史记录"标题栏展开/收起，带展开动画
  - **按日期倒序**: 展开后按日期倒序展示每天的打卡记录，每天一个卡片
  - **每日汇总**: 每个日期卡片头部显示日期、当日奖励总额、惩罚总额、当日净值
  - **打卡明细**: 每个日期卡片内列出当天的打卡条目（事项名称、完成状态图标、金额变化），已完成显示绿色勾、未完成显示红色叉
  - **空状态**: 无历史记录时显示"暂无打卡记录"

#### 区块: 底部悬浮操作栏

- **页面目标**: 提供快捷操作入口
- **功能点**:
  - **添加任务按钮**: 主按钮（+），点击弹出添加任务 Dialog
  - **数据管理按钮**: 次按钮（菜单图标），点击弹出操作菜单（导出数据 / 导入数据 / 永久删除事项管理）
  - **固定悬浮**: 移动端固定在底部，桌面端固定在右下角，不随页面滚动遮挡内容

## 弹窗/抽屉功能

### 添加/编辑任务 Dialog

- **触发**: 底部操作栏"添加任务"按钮 / 任务列表事项右侧编辑图标
- **功能点**:
  - **表单字段**: 事项名称（文本输入）、金额（数字输入，正数）、类型（奖励/惩罚 切换开关）、重复类型（每日重复/一次性 单选）、开始日期（日期选择器）、结束日期（日期选择器，可选，不选则无限期）
  - **表单验证**: 名称为必填，金额必须 > 0，开始日期不能晚于结束日期
  - **提交**: 新增时创建事项（生成唯一 ID、记录创建时间），编辑时更新事项字段，保存后同步到云端数据库，关闭 Dialog，刷新任务列表
  - **编辑模式**: 编辑已有事项时，表单预填现有数据，标题显示"编辑事项"

### 删除/恢复事项管理

- **触发**: 任务列表事项右侧删除图标 / 底部操作栏菜单中的"管理已删除事项"
- **功能点**:
  - **软删除**: 删除事项时，将事项标记为 `deleted: true`（而非物理删除），保留打卡记录
  - **恢复事项**: 在"已删除事项"列表中展示所有软删除的事项，点击恢复按钮将 `deleted` 标记移除，事项重新出现在任务列表中
  - **永久删除**: 在"已删除事项"列表中提供"永久删除"按钮，点击后弹出确认 Dialog，确认后物理删除事项数据 + 清除该事项所有关联的打卡记录，不可恢复

### 数据导入 Dialog

- **触发**: 底部操作栏菜单中的"导入数据"
- **功能点**:
  - **文件选择**: 点击后触发文件选择器，仅接受 `.json` 文件
  - **数据解析**: 读取 JSON 文件内容，校验数据结构（必须包含 items、records、initialBalance 字段）
  - **覆盖确认**: 解析成功后弹出确认 Dialog，提示"导入将覆盖现有所有数据，是否继续？"
  - **执行导入**: 确认后用导入数据覆盖当前应用状态，同步到云端数据库，显示导入成功 toast

### 数据导出

- **触发**: 底部操作栏菜单中的"导出数据"
- **功能点**:
  - **数据组装**: 将当前所有数据（事项列表、打卡记录、初始余额、用户标识）组装为 JSON 对象
  - **文件下载**: 生成 JSON 文件并触发浏览器下载，文件名格式 `reward-tracker-backup-YYYYMMDD-HHmmss.json`
  - **反馈**: 显示"导出成功" toast

## 数据共享配置

| 存储键名 | 数据说明 | 使用页面 |
|---------|---------|---------|
| `__tracker_user_id` | 当前用户标识（用户名/同步码），类型 `string` | 用户标识页、主打卡页 |
| `__tracker_offline_cache` | 离线数据缓存（事项列表 + 打卡记录 + 初始余额），类型 `ICacheData`，在线时优先云端、离线时降级使用 | 主打卡页 |

```ts
interface ICacheData {
  userId: string;
  items: IItem[];
  records: IRecordMap;
  initialBalance: number;
  lastSyncAt: string; // ISO 时间戳
}

interface IItem {
  id: string;
  name: string;
  amount: number;
  isReward: boolean; // true=奖励, false=惩罚
  repeatType: 'daily' | 'once'; // 每日重复 / 一次性
  targetDate?: string; // 目标日期（一次性事项使用）
  startDate: string; // 开始日期 YYYY-MM-DD
  endDate?: string; // 结束日期 YYYY-MM-DD，不设则无限期
  createdAt: string; // ISO 时间戳
  deleted?: boolean; // 软删除标记
}

interface IRecordMap {
  [date: string]: IEntry[]; // key 为 YYYY-MM-DD 格式日期
}

interface IEntry {
  itemId: string;
  itemName: string;
  completed: boolean; // true=完成(奖励)/未完成(惩罚), false=未完成(奖励)/完成(惩罚)
  amountChange: number; // 金额变化（正数为增加，负数为扣除）
  timestamp: string; // ISO 时间戳
}
```

## 余额计算逻辑说明

> **说明**：以下计算逻辑为前端实时计算，不依赖后端。

**当前余额** = `initialBalance` + Σ(所有已打卡奖励事项的 `amount`) - Σ(所有已打卡惩罚事项的 `amount`)

**今日收益** = 今日奖励总额 - 今日惩罚总额

**累计获得奖励总额** = Σ(所有历史已打卡奖励事项的 `amount`)

**累计惩罚总额** = Σ(所有历史已打卡惩罚事项的 `amount`)

**净收益** = 累计获得奖励总额 - 累计惩罚总额

**余额趋势图数据**（7天/30天）:
- 以当前选中日期为终点，向前取 N 天
- 每天计算当日结束时的余额 = 前一天余额 + 当日奖励 - 当日惩罚
- 第一天余额 = `initialBalance` + 第一天奖励 - 第一天惩罚
- 生成 `[{date: 'YYYY-MM-DD', balance: number}, ...]` 数组喂给折线图

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free Direction —— 无参考材料，从产品语义与使用场景自主建立视觉方向
- **核心情绪 / 应用类型**: 个人成长记账工具 —— 奖励是正向激励、惩罚是自我约束，界面需要传递轻盈、可信、有仪式感的日常记录氛围
- **独特记忆点**: 余额卡片作为"个人账本封面"，用柔和的渐变光晕和数字跳动感，让每次打卡都像在翻一页属于自己的成长存折

## 2. Art Direction

- **方向名**: Soft Ledger · 柔和账本
- **Design Style**: Rounded 圆润几何 + Warm Natural 自然暖调 —— 奖励打卡不是冷冰冰的财务工具，而是带有温度的个人成长伴侣；圆润的卡片和暖色基调降低打卡压力，让自律变得可亲近
- **DNA 参数**: 圆角 soft（`rounded-xl` ~ `rounded-2xl`）/ 阴影 subtle（`shadow-sm`，余额卡片用柔光扩散）/ 间距 standard（`gap-4` `p-5`）/ 字体方向 清晰无衬线为主，数字用等宽增强记账感 / 装饰手法 极简，靠色彩分层和卡片投影建立空间，不用多余图标和分割线
- **应用类型**: Tool —— 手机优先的任务打卡工具，桌面端自适应

## 3. Color System

**色彩关系**: 暖琥珀主色 + 同色极浅奶油反馈底 + 暖白纸张背景 + 冷翡翠绿作为正向奖励语义色
**配色设计理由**: primary 选用暖琥珀色，传递"收获"和"积累"的情绪，比冷蓝色更贴近个人记账场景；bg 用暖白纸张色，让每日打卡有翻开笔记本的熟悉感；奖励用翡翠绿表达正向增长，惩罚用柔和珊瑚红表达扣减但不制造焦虑
**主色推导**: 从"奖励/余额/成长"语义出发，琥珀色天然带有金币、收获、温暖的联想，比金融蓝更贴合个人激励工具；饱和度控制在 60-70%，避免过于浓烈
**使用比例**: 60% 中性暖白 / 30% 辅助浅琥珀底 / 10% primary 琥珀主行动

**语义色提示**:
- 奖励/正向增长: `hsl(155 45% 48%)` — bg `hsl(150 40% 94%)` / border `hsl(155 35% 80%)` / text `hsl(155 45% 48%)`；翡翠绿与琥珀主色形成暖-冷互补，饱和度对齐 ±10%
- 惩罚/扣减: `hsl(8 55% 58%)` — bg `hsl(8 45% 94%)` / border `hsl(8 40% 82%)` / text `hsl(8 55% 58%)`；柔和珊瑚红，不制造警告感，饱和度与 primary 对齐
- 余额趋势图: 奖励用翡翠绿 `hsl(155 45% 48%)`，惩罚用珊瑚红 `hsl(8 55% 58%)`，净值线用 primary 琥珀 `hsl(35 65% 52%)`

## 4. 字体与节奏

- **font-display**: Noto Sans SC —— 中文清晰可读，字重覆盖全，适合移动端卡片标题和余额数字
- **font-body**: Inter —— 数字和英文部分用 Inter，其等宽数字变体（`font-feature-settings: "tnum"`）让余额、金额列对齐，增强记账感
- **字号**: H1（余额大数字）text-4xl ~ text-5xl；H2（卡片标题）text-xl ~ text-2xl；body text-sm ~ text-base；muted text-xs ~ text-sm
- **圆角**: soft — `rounded-xl` 卡片、`rounded-2xl` 余额大卡片、`rounded-lg` 按钮和输入框；圆角统一偏大，传递柔和触感

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导 — 手机优先单列布局，桌面端双列（左侧打卡区 + 右侧历史/趋势区）
- **Page / Section Order**: 余额大卡片 → 日期导航栏 → 任务打卡列表 → 余额趋势图（可折叠）→ 历史记录（可折叠）→ 底部悬浮操作栏
- **Standard Content Zone**: 移动端 `max-w-lg`，桌面端 `max-w-5xl`，均 `mx-auto`
- **Shell / Frame Alignment**: 内容区与框架同宽，无侧边栏；顶部固定用户头像/同步码入口，底部固定悬浮操作栏
- **Padding & Rhythm**: 移动端 `px-4 py-4`，桌面端 `px-6 py-6`；卡片间距 `gap-4`，段落间距 `space-y-4`
- **Full-bleed Zones**: 余额大卡片背景可做柔和渐变光晕延伸至 `w-full`，内部文字和按钮仍受 `max-w-lg` / `max-w-5xl` 约束
- **Local Narrowing**: 设置页、用户切换页、数据导入导出页在桌面端收窄至 `max-w-2xl`
- **Overflow Strategy**: 历史记录列表和趋势图使用 `overflow-y-auto` 局部滚动，不撑开全局高度
- **Flexibility Boundary**: 允许移动端调整卡片内边距和按钮尺寸；全局 max-w、圆角系统、主色和阴影语言保持一致

## 6. 视觉与动效

- **装饰**: 余额卡片顶部用极淡的琥珀到奶油色径向渐变光晕（`radial-gradient`），模拟柔和聚光灯打在账本封面上；无多余图标和分割线
- **阴影/边界**: 轻 — 卡片 `shadow-sm`，余额大卡片用 `shadow-md` + 微弱 `blur` 扩散，制造"浮起"的封面感；打卡按钮用边界 `border` 区分完成/未完成状态
- **动效**: 精致 — 打卡按钮点击有轻微缩放（`scale-95` 按压态）+ 完成态颜色过渡（`transition-colors duration-200`）；余额数字变化用 `transition-all` 数字跳动；列表项入场用 `fade-in` + 上移 8px 的 stagger 动画；日期切换用横向滑动过渡

## 7. 组件原则

- 打卡按钮: Default 为暖灰边框空心圆（未完成），Active 为翡翠绿实心圆 + 对勾图标（已完成），按压 `scale-95`，Focus 有琥珀色 `ring-2` 偏移
- 余额大卡片: 白色卡片 + 顶部暖光渐变，数字用 `font-feature-settings: "tnum"` 等宽对齐，今日净收益用翡翠绿或珊瑚红小字标注
- 日期导航: 左右箭头用 accent 浅底 + primary 图标色，点击日期弹出 mini 日历，选中日用 primary 实心圆标记
- 底部悬浮栏: 白色卡片 + `shadow-lg`，圆角 `rounded-t-2xl`，包含"添加事项"主按钮（primary 实心）和"撤销打卡"次按钮（outline）
- 空状态: 首次无任务时，卡片内显示柔和插画 + "开始你的第一条打卡"引导文字，按钮用 primary
- 加载与同步: Skeleton 用 accent 底色 + 微弱脉冲动画；同步中在顶部显示细条琥珀色进度指示

## 8. Image Direction

- **Image Role**: 空状态插画 — 用于首次进入应用、无任务、无历史记录时的引导占位
- **Image Art Direction**: 扁平插画风格，暖白底色上放置一本翻开的笔记本，书页上浮起几枚柔和琥珀色的小星星或硬币符号，旁边有一支圆润的笔；光线从左上角柔和洒下，整体传递"开始记录你的成长"的安静仪式感；无人物、无复杂场景，构图居中留白
- **Image Prompt Keywords**: flat illustration, open notebook on warm cream background, floating amber coin icons, soft rounded pen, gentle top-left lighting, minimalist, cozy personal journal vibe, no characters, centered composition with breathing space
- **Image Avoidance**: 避免通用商务人物插画、握手/图表/火箭等企业素材符号、过于卡通的夸张表情、高饱和渐变背景、3D 拟物化金币堆叠

## 9. Anti-patterns

- **Split personality**: 余额卡片用柔和光晕但任务列表用冷白直角卡片；全站统一圆角系统和暖白基调
- **Phantom tokens**: 编造 `--success` / `--danger` 等 shadcn 未定义变量；语义色通过 Tailwind 自定义 class 或 `style` 属性注入
- **Default SaaS drift**: 回到默认蓝色主按钮和紫色渐变；坚持琥珀主色 + 翡翠绿奖励 + 珊瑚红惩罚的语义配色
- **Invisible interaction**: 打卡按钮只有颜色变化没有 `focus-visible` 环；每个可交互元素都要有琥珀色 `ring-2` 键盘焦点态
- **Mono-hue tyranny**: 主按钮、tab、图标、边框全用琥珀色；按 60-30-10 把 primary 收回 CTA 和余额高亮，边框和辅助态用 accent 暖灰
- **Status color drift**: 奖励绿和惩罚红饱和度过高，与克制琥珀主色脱节；语义色饱和度与 primary 对齐在 45-65% 区间
- **Mobile-first neglect**: 桌面端卡片过宽导致打卡按钮和文字间距失控；保持 `max-w-lg` 移动端优先，桌面端双列时每列仍受合理宽度约束
